# FundThePitch 系統啟用流程

## 角色說明

| 角色 | 說明 |
|------|------|
| **最高管理員（super_admin）** | 系統最高權限，可跨場次管理所有資料 |
| **系所管理員（admin）** 系秘 | 由最高管理員建立，負責管理單一系所的專題發表會 |
| **評審（judge）** | 由系秘建立，進入指定會場後進行模擬投資評分 |

---

## 啟用流程

### Step 1：最高管理員建立系秘帳號

1. 最高管理員登入系統，進入 **管理員 → 成員管理**。
2. 在「新增成員」欄位輸入 **系所秘書的姓名**，選擇角色為「管理者（admin）」，按下新增。
3. 系統以輸入的姓名作為帳號識別碼，直接建立帳號，無需額外設定密碼。
4. 最高管理員將帳號資訊（姓名即帳號）告知系秘，**此後最高管理員前置作業完成，暫無後續操作**。

---

### Step 2：系秘登入並啟動專題會（場次）

1. 系秘使用帳號登入系統。
2. 進入 **管理員 → 場次管理**，在「場次名稱」欄位輸入本次活動名稱，例如：
   ```
   資管系專題發表會
   ```
3. 按下 **啟動新場次**。
4. 系統自動產生 **邀請網址** 與 **QR Code**。
5. 複製邀請網址 / 列印 QR Code，**置於各評審桌前**，供評審當天掃描或點擊連結進入對應場次。

> ⚠️ 系秘每次只能同時維護一個進行中的場次，無需選擇場次的額外操作。

---

### Step 3：系秘設定會場（會議廳）

1. 進入 **管理員 → 會場管理**。
2. 在「新增會場」欄位輸入會場教室名稱（例如 `LM402`），按下 **新增**。
3. 點擊剛建立的會場卡片，進入 **修改會場** 設定頁，依序填寫：

   | 欄位 | 說明 |
   |------|------|
   | **教室名稱** | 已預填，可修改（例如 `LM402`） |
   | **專題組（每行一筆）** | 輸入此會場所有專題組名稱，一行一筆；亦可掃描 XML 檔批次匯入 |
   | **從今年度成員加入評審** | 從下拉選單點選本場次評審，多人可逐一選取，已選者會顯示為標籤 |
   | **新增成員並直接加入這個會場（每行一位姓名）** | 若評審帳號尚未建立，可在此直接輸入姓名（一行一位），系統會自動建立帳號並指派到此會場 |

4. 填寫完畢按下 **完成會場設定**。
5. 重複步驟 2–4，直到所有會場都設定完成。

---

### Step 4：前置作業完成

- 所有會場與評審設定完成後，**系秘的前置作業即告結束**。
- 活動當天，評審透過桌前的 QR Code / 網址進入系統，系統會自動導引至其指派會場，無需手動加入。

---

## 備注

- 最高管理員在 **會場管理** 頁面可看到「選擇場次」選單，用於跨場次管理；系秘登入後此選單不顯示，以避免誤操作。
- 邀請網址與 QR Code 可在「場次管理」頁面隨時重新取得。
- 若評審尚未投票，系秘可在「成員管理」中隨時調整會場指派或刪除帳號。
